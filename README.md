# CRUD (Postgres + Redis)

Демонстрационный стенд, который показывает, как связка PostgreSQL + Redis ускоряет типичные операции CRUD (создание, чтение, обновление, удаление) и разгружает базу данных.

## Зачем это нужно

- **Мгновенные ответы.** Redis хранит часто запрашиваемые данные в памяти. Повторные запросы отдаются практически без задержек.  
- **Меньше нагрузка на Postgres.** Популярные записи читаются из кэша, а база данных используется для надёжного долгосрочного хранения.  
- **Актуальный кэш.** При создании, изменении или удалении пользователя мы сразу обновляем или очищаем соответствующий ключ в Redis, поэтому данные не расходятся.  
- **Готовность к боевому использованию.** Redis легко расширить под хранение сессий, rate-limiting, очереди сообщений, pub/sub и многое другое.  
- **Наглядность.** Ключи вида `users_add:{id}` позволяют увидеть, как данные из базы появляются в кэше и живут там ограниченное время.

## Быстрый старт

1. Перейдите в каталог стенда:
   ```bash
   cd crud(postgres+redis)
   ```
2. Поднимите сервисы:
   ```bash
   docker compose up --build
   ```
3. Откройте браузер и перейдите по адресу  
   `http://localhost:5001`.

### Порты

| Сервис    | Контейнер | Хост |
|-----------|-----------|------|
| Flask-приложение | 5001 | 5001 |
| Postgres  | 5432      | 5433 |
| Redis     | 6379      | 6379 |

## Как зайти в Redis

### Через docker-compose

```bash
docker compose exec redis redis-cli KEYS "*"
```
